I wanted to create an application that would be simplistic in nature. The project idea that I came up with is a mobile application that lets people make cocktails, mixed drinks, mocktails etc. in the comfort of their own home. The application allows you to put in drink preferences, what beverages the user has access to, and from there provides the what beverages the user is able to make with what they have on hand, the recipes for said beverages and also what they would be able to make with if they had a few more ingredients.

I wanted to choose this for my project as I am someone who enjoys a good cocktail or beverage, and through personal experience, I don’t really know what drinks I can make with what I have available currently, so that’s why I wanted to create an application that would be able to show users what’s available at their fingertips. Statistics from the National Health Survey also found that in 2017–18 among Australians aged 18 and over, 79% had consumed alcohol in the past year, so an application that would allow users to make cocktails with what’s available at home would be something desirable, especially during the current pandemic and lockdowns, where many people are unable to access a good cocktail, mocktail or mixed drink.

Now I’m going to be going into more specifics about the features of the application. The application would be quite bare-boned, hopefully having a minimalistic and clean feel to it. Users are able to create accounts on the application if they would want to save recipes and/or want to have available ingredients automatically inserted each time they open the application. If not, the application would be able to accessed through a guest mode, just for ease of access for those that do not want to go through a sign up process or create an account. Once the application is accessed, it should be easy to locate what the users are looking for, with a bottom or top bar showing settings, account, ingredients, recipes and saved recipes. A feature that the application would have that I believe would be very helpful is a section where history of viewed recipes would be accessible, as often, many people look at something and forget to save it for later, causing inconvenience. In the settings, users are able to change app features such as to hide alcoholic recipes if for underage users and other settings to enhance user experience like notifications, a help centre, themes and whether the user would prefer imperial or metric measurements. In the account tab, users are able to add friends and/or follow people to see what beverages other people are making and perhaps take inspiration. To create an account they can connect via third parties such as Facebook and Google, among other parties. Users are also able to rate recipes they have previously created which would show on profile and would be able to be public or private depending on user settings. The ingredients tab would allow for users to insert what ingredients they have readily available at home from a drop down menu depending on alcohol (if applicable), mixers, and further ingredients. If something is not available, users are able to input an ‘Other: ‘ option where the system would search for that ingredient via name. In the recipes tab, there filter selector, where users are able to select whether they would want a mild, medium, or strong alcohol beverage, if they want to show recipes where the user would have all but a few ingredients and what type of beverage they would like. The recipes tab would then show the users what recipes are available to them, with pictures, main ingredients and ratings from other users. If no beverages are able to be made with what the user has available then beverages that are able to be made with a few more ingredients are shown. Saved recipes tab would show all the recipes the user has saved for later, the tab also would have a feature where the user is able to group and filter the beverages they have saved, having the ability to create folders for beverages. The overall service of the application would be easily accessible without flashy features to confuse the user.
